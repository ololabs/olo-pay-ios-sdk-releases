Changelog

v2.1.2 (Nov 15th, 2022)

  • Changed Podspec to use ssh instead of https
  • Fixed missing setup guides in documentation

v2.1.1 (Nov 9, 2022)

  • Added missing podspec file to fix CocoaPods usage
  • Updated Carthage usage guide with proper tag syntax

v2.1.0 (Nov 7, 2022)

  • Added Carthage support (Note: Build path of Stripe dependencies changed)
  • Added CocoaPods support
  • Add US and CA postal code validation to OPPaymentCardDetailsView

v2.0.0 (Sep 27, 2022)

  • Breaking Change: Simplified SDK setup and initialization
    • Removed OloPaySDK-Dev target
    • Added OPEnvironment enum
    • Added environment parameter and reordered existing parameters on OPSetupParams
  • Update to Stripe iOS SDK v22.8.1
  • Fixed Xcode 14 compilation error
  • Fixed OPPaymentCardDetailsView postal code error message displaying when it shouldn’t

v1.2.1 (May 18, 2022)

  • Error message now displays for OPPaymentCardDetailsView if getPaymentMethodParams() is called and card details are invalid
  • Added OPPaymentCardDetailsView.hasErrorMessage(...)
  • Added OPPaymentCardDetailsView.getErrorMessage(...)
  • Added OPCardBrand.unsupported
  • Updated card error messages to distinguish between invalid card numbers and unsupported card numbers
  • Added “Log Form Valid Changes” option to Test Harness settings
  • Fixed typo in error message for empty CVC fields
  • Fixed test harness incorrectly logging whether the OPPaymentCardDetailsForm is valid or not
  • Removed Carthage files to reduce SDK download size

1.2.0 (Feb 28, 2022)

  • Update to Stripe iOS SDK v21.12.0
  • Test Harness compilation fixes after updating Stripe SDK
  • Fixes for hybrid app solutions (e.g. React Native)
  • Official React Native bridging files support
  • Added SKD setup steps to documentation

1.1.3 (Dec 17, 2021)

  • OloPayAPI changes
    • Added safeguard to OloPayAPI.createPaymentRequest(...) for Apple Pay merchant id and company name
    • Added more robust retry mechanism in OloPayAPI.createPaymentMethod(...)
  • Added completion handler to OloPayApiInitializer.setup(...)
  • Added public constructor to OPError
  • Added country property to OPPaymentMethod.country and OPPaymentMethodProtocol.country
  • Added unit tests

1.1.2 (Oct 22, 2021)

  • Minor Xcode project tweaks

1.1.1 (Oct 22, 2021)

  • Minor Xcode project tweaks

1.1.0 (Oct 21, 2021)

  • Fixed issue with Carthage always pulling the latest version of the Stripe SDK
  • Added missing documentation
  • New Classes/Protocols/Enums
    • Added OloPayApiInitializer
    • Added OloPayAPIProtocol
    • Added OPPaymentCardDetailsForm
    • Added OPPaymentMethodProtocol
    • Added OPPaymentMethodParams
    • Added OPPaymentMethodParamsProtocol
    • Added OPPaymentCardDetailsForm
    • Added OPCardFormStyle
    • Added OPStrings
  • Removed OloPayAPIGateway… use OloPayAPI directly
  • OloPayAPI changes
    • Moved setup() to OloPayApiInitializer
    • Added createPaymentMethod(...) that takes an instance of OPPaymentMethodParamsProtocol
    • Deprecated createPaymentMethod(...) that takes an instance of OPPaymentCardDetailsForm
    • Deprecated createPaymentMethod(...) that takes an instance of OPPaymentCardDetailsView
    • More robust error handling in OloPayAPI.createPaymentMethod(...)
  • OPPaymentCardDetailsView changes
    • Added missing @objc annotations for better obj-c support
    • Added OPPaymentCardDetailsView.expirationIsValid convenience property
    • Added OPPaymentCardDetailsView.expirationIsEmpty convenience property
    • Added OPPaymentCardDetailsView.postalCodeIsEmpty property
    • Error message enhancements
      • Added ability to provide custom error messages via OPPaymentCardDetailsView.errorMessageHandler
      • Added ability to turn off display of error messages
      • Added OPPaymentCardDetailsView.errorMessage property
      • Added OPPaymentCardDetailsViewgetUserFacingMessage(...) to get an error message for a specific field
    • Added OPPaymentCardDetailsView.getPaymentMethodParams(...)

1.0.1 (Sep 24, 2021)

  • OPPaymentCardDetailsView changes
    • OPPaymentCardDetailsView.isValid now returns false if the card type isn’t supported by Olo Pay
    • New properties to check the validity of each card field
  • OloPayAPI.createPaymentMethod(...) returns an error if the card type isn’t supported by Olo Pay
